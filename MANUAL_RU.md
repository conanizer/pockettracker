## 1. Введение

PocketTracker — это семпловый музыкальный трекер для Android-консолей и бюджетных Android-устройств. Приложение вдохновлено трекером M8 Headless и Little GP Tracker (LGPT/Picotracker), и разработано для работы в разрешении **640×480** — как у устройств типа Miyoo Flip.

**Что такое трекер?** Трекер — это музыкальный секвенсор, в котором ноты записываются в сетку: время движется сверху вниз, каждая строка — это шаг, каждый столбец — параметр (нота, инструмент, громкость, эффект). Песни строятся из небольших паттернов — **фраз** (phrase), которые объединяются в **цепочки** (chain), а цепочки расставляются в **песне** (song).

PocketTracker хранит всё в одном файле проекта (`.ptp`). Звуки загружаются из стандартных файлов `.wav`.

### Поддерживаемые устройства

| Устройство | ОЗУ | ОС | Управление |
|---|---|---|---|
| Miyoo Flip | 1 ГБ | Android 13 | Физические кнопки |
| Ayaneo Pocket Air Mini | 3 ГБ | Android 11 | Физические кнопки |
| Любой Android 8.0+ телефон/планшет | 512 МБ+ | Android 8+ | Сенсорный экран |

## 2. Установка

1. Скачайте последний `.apk`-файл со страницы релизов проекта.
2. На устройстве Android включите **Установку из неизвестных источников** в Настройках → Безопасность.
3. Откройте скачанный `.apk` и нажмите **Установить**.
4. При первом запуске приложение может запросить разрешение **Доступ ко всем файлам** (Android 11+). Разрешите — это необходимо для работы с проектами и семплами.

### Файлы семплов

Стандартные семплы (бочка, малый барабан, хэт, бас и т. д.) встроены в приложение. Чтобы использовать собственные `.wav`-файлы, скопируйте их в память устройства и загрузите через экран **INSTRUMENT** с помощью файлового браузера.

### Файлы проектов

Проекты сохраняются как `.ptp`-файлы в:

```
/Documents/PocketTracker/Projects/
```

WAV-экспорт сохраняется в:

```
/Documents/PocketTracker/Renders/
```

Ресемплированные инструменты сохраняются в:

```
/Documents/PocketTracker/Samples/Resampled/
```

## 3. Обзор интерфейса

Весь интерфейс отрисовывается на фиксированном холсте **640×480** пикселей, с полосами-рамками (letterbox) на больших экранах. Расположение элементов одинаково на всех экранах.

```
┌────────────────────────────────────────────────┐
│  ОСЦИЛЛОГРАФ  (620×70 пк)                      │
├──────────────────────────────┬─────────────────┤
│                              │  КАРТА ЭКРАНОВ  │
│  ОСНОВНОЙ РЕДАКТОР           │  (80×105 пк)    │
│  (зависит от экрана)         │                 │
│                              │  СТРОКА СТАТУСА │
│                              │                 │
└──────────────────────────────┴─────────────────┘
```

**Осциллограф** — отображает форму волны смешанного аудио в реальном времени. Всегда виден. Прокручивается справа налево.

**Основной редактор** — активный экран (PHRASE, CHAIN, SONG и т. д.).

**Карта экранов** — миниатюрная сетка 5×5, показывающая вашу текущую позицию в навигации.

**Строка статуса** — короткие сообщения (например, «SAVED», «RESAMPLED TO INST 0C»), которые автоматически исчезают через 5 секунд.

## 4. Навигация

Все экраны расположены в **сетке 5×5**. Навигация по сетке осуществляется удержанием **R** и нажатием крестовины.

```
     Кол 0      Кол 1      Кол 2      Кол 3      Кол 4
     ─────      ─────      ─────      ─────      ─────
Ряд 0  ---        ---       SCALE    INST POOL    ---
Ряд 1  PROJ       PROJ      GROOVE     MODS       ---
Ряд 2  SONG      CHAIN     PHRASE     INST       TABLE
Ряд 3  MIXER     MIXER     MIXER      MIXER      MIXER
Ряд 4  EFFECTS  EFFECTS   EFFECTS   EFFECTS    EFFECTS
```

- **R + ВПРАВО / ВЛЕВО** — движение влево/вправо по 2-му ряду (основные экраны редактирования)
- **R + ВВЕРХ / ВНИЗ** — переход на экран выше или ниже в текущем столбце
- Карта экранов в правом верхнем углу всегда показывает текущее положение

> **Подсказка:** Самый частый маршрут — PHRASE (кол 2) ↔ INSTRUMENT (кол 3) ↔ TABLE (кол 4), всё в ряду 2. Нажмите **R+ВВЕРХ** из любого из них, чтобы попасть на контекстные экраны (GROOVE, MODS).

## 5. Управление

### 5.1 Раскладка кнопок

#### Физический геймпад (Android-консоли)

| Физическая кнопка | Функция |
|---|---|
| Крестовина (D-pad) | Перемещение курсора |
| A | Подтверждение / Вставка |
| B | Отмена / Удаление |
| L1 / L2 | Модификатор L |
| R1 / R2 | Модификатор R |
| SELECT / MENU | Модификатор SELECT |
| START / BACK | Воспроизведение / Остановка |

#### Клавиатура (для тестирования на ПК или через Bluetooth-клавиатуру)

| Клавиша | Функция |
|---|---|
| W / S / A / D или стрелки | Крестовина |
| K или Enter | Кнопка A |
| J или Escape | Кнопка B |
| U | Кнопка L |
| I | Кнопка R |
| Left Shift | SELECT |
| Пробел | START |

Клавиатура и геймпад работают одновременно.

---

### 5.2 Основные действия

| Ввод | Действие |
|---|---|
| Крестовина | Перемещение курсора |
| A | Вставить значение (на пустой ячейке вставляет последнее использованное) |
| B | Удалить значение / отмена |
| SELECT | Контекстное удаление (зависит от экрана) |
| START | Воспроизведение / Остановка |

---

### 5.3 Редактирование значений — A + Крестовина

Удерживайте **A** и нажимайте направление для редактирования значения под курсором:

| Комбинация | Шаг |
|---|---|
| A + ВВЕРХ | +1 (малый шаг) |
| A + ВНИЗ | −1 (малый шаг) |
| A + ВПРАВО | +16 / +1 октава (большой шаг) |
| A + ВЛЕВО | −16 / −1 октава (большой шаг) |
| A + B | Удалить / очистить значение |

- **Автоповтор активен:** удерживайте комбинацию ~400 мс — начинается повтор со скоростью ~10 раз/сек.
- Для **нотных значений** большой шаг = ±12 полутонов (одна октава).
- Для **шестнадцатеричных байтов** большой шаг = ±0x10 (±16).

---

### 5.4 Контекстная навигация — L + Крестовина

Удерживайте **L** и нажимайте ВЛЕВО/ВПРАВО для навигации между элементами одного типа:

| Экран | L + ВЛЕВО / ВПРАВО |
|---|---|
| CHAIN | Предыдущая / следующая цепочка (00–FF) |
| PHRASE | Предыдущая / следующая фраза (00–FF) |
| INSTRUMENT | Предыдущий / следующий инструмент (00–FF) |
| TABLE | Предыдущая / следующая таблица (00–FF) |
| GROOVE | Предыдущий / следующий грув (00–FF) |

---

### 5.5 Навигация по экранам — R + Крестовина

Удерживайте **R** и нажимайте направление для перемещения по сетке экранов:

| Комбинация | Движение |
|---|---|
| R + ВЛЕВО / ВПРАВО | Влево / вправо (остаётся в текущем ряду) |
| R + ВВЕРХ | Экран выше текущего столбца |
| R + ВНИЗ | Экран ниже текущего столбца |

---

### 5.6 Копирование / Вставка

PocketTracker использует систему выделения в стиле M8. Работает на экранах PHRASE, CHAIN, SONG и TABLE.

| Ввод | Действие |
|---|---|
| L + B | Войти в режим выделения (повторное нажатие переключает: CELL → ROW → SCREEN) |
| B (в режиме выделения) | Скопировать выделение, выйти из режима |
| L + A (в режиме выделения) | Вырезать (копировать + удалить), выйти из режима |
| L + A (вне режима выделения) | Вставить из буфера обмена в позицию курсора |
| A + B (в режиме выделения) | Удалить выделение (без копирования), выйти из режима |
| L (без других кнопок) | Отменить выделение (ничего не копируется) |

**Режимы выделения:**
- **CELL** — одна ячейка под курсором
- **ROW** — целая строка (все столбцы)
- **SCREEN** — все видимые строки

---

### 5.7 Управление воспроизведением

| Ввод | Действие |
|---|---|
| START | Воспроизведение / Остановка (в зависимости от текущего экрана) |
| START на SONG | Воспроизведение полной песни |
| START на CHAIN | Воспроизведение текущей цепочки |
| START на PHRASE | Воспроизведение текущей фразы (зациклено) |
| START на INSTRUMENT | Предпрослушивание текущего семпла |

## 6. Структура песни

PocketTracker организует музыку в виде четырёхуровневой иерархии:

```
ПРОЕКТ (PROJECT)
  └── ПЕСНЯ (SONG) — 8 дорожек, в каждой — последовательность цепочек
        └── ЦЕПОЧКА (CHAIN) — до 16 ссылок на фразы + транспонирование
              └── ФРАЗА (PHRASE) — 16 шагов
                    └── ШАГ (STEP) — Нота + Инструмент + Громкость + 3 слота эффектов
```

Как это работает:

- **ШАГ** — одно нотное событие с опциональными эффектами.
- **ФРАЗА** — короткий паттерн из 16 шагов, как один такт.
- **ЦЕПОЧКА** — последовательность до 16 фраз, воспроизводимых одна за другой. Каждый слот может иметь значение **транспонирования** — это позволяет играть фразу другой высотой без дублирования.
- **ПЕСНЯ** расставляет цепочки по 8 дорожкам в вертикальном списке. Каждая строка песни воспроизводит все 8 дорожек одновременно.

Все значения (ID цепочек, фраз, инструментов и т. д.) — шестнадцатеричные, от `00` до `FF`.

## 7. Экран SONG (Песня)

Экран SONG расставляет цепочки по 8 дорожкам. Каждый столбец — дорожка (T0–T7), каждая строка — позиция в песне.

```
     T0   T1   T2   T3   T4   T5   T6   T7
00   04   --   08   --   --   01   --   --
01   04   --   08   --   --   01   --   --
02   05   --   09   --   --   02   --   --
...
```

- `--` означает, что дорожка молчит на данной позиции.
- Числа — ID цепочек (в шестнадцатеричном формате).

### Управление

| Ввод | Действие |
|---|---|
| Крестовина | Перемещение курсора |
| A | Вставить последнюю использованную цепочку |
| A + ВВЕРХ/ВНИЗ | Увеличить / уменьшить ID цепочки |
| A + ВПРАВО/ВЛЕВО | Изменить на ±16 |
| A + B | Удалить (установить в --) |
| START | Воспроизведение песни с начала |

---

## 8. Экран CHAIN (Цепочка)

Цепочка — это последовательность до 16 ссылок на фразы. Каждый слот содержит:
- **PHR** — ID фразы (00–FF) или `--` (пусто/тишина)
- **TRN** — транспонирование (+/−) в полутонах (00 = без транспонирования; значения выше 7F — отрицательные)

```
     PHR  TRN
00   04   00
01   04   00
02   05   0C   ← +12 полутонов (одна октава вверх)
03   05   00
...
```

При воспроизведении цепочка зациклена: после последнего заполненного слота возвращается к слоту 00.

### Управление

| Ввод | Действие |
|---|---|
| Крестовина | Перемещение курсора |
| A | Вставить последнее использованное значение |
| A + ВВЕРХ/ВНИЗ | Увеличить / уменьшить |
| A + ВЛЕВО/ВПРАВО | ±16 (или ±12 полутонов для TRN) |
| A + B | Удалить слот |
| L + ВЛЕВО/ВПРАВО | Предыдущая / следующая цепочка |
| START | Воспроизведение текущей цепочки |

---

## 9. Экран PHRASE (Фраза)

Фраза — место, где пишутся ноты. Содержит 16 строк (шаги 00–0F) и 5 столбцов:

```
     N    V    I    FX1      FX2      FX3
00   C-4  80   03   ---  00  ---  00  ---  00
01   ---  --   --   ---  00  ---  00  ---  00
02   E-4  80   03   ---  00  ---  00  ---  00
...
```

| Столбец | Значение |
|---|---|
| N | Нота (например, `C-4`, `F#3`). `---` = нет ноты. |
| V | Громкость (`00`–`FF`). `--` = использовать значение инструмента. |
| I | ID инструмента (`00`–`FF`). `--` = использовать последний инструмент. |
| FX1/FX2/FX3 | Тип эффекта + значение (например, `REP 03`, `ARP 47`) |

### Ноты

Ноты записываются как высота + октава: `C-4`, `C#4`, `D-4`, … `B-9`. Диапазон: **C-0 до B-9**.

Средняя C — `C-4` (MIDI-нота 60).

### Управление

| Ввод | Действие |
|---|---|
| Крестовина | Перемещение курсора |
| A | Вставить последнее использованное значение ноты |
| A + ВВЕРХ/ВНИЗ | +1 / −1 полутон (нота), +1 / −1 (другие значения) |
| A + ВЛЕВО/ВПРАВО | +1 / −1 октава (нота), +16 / −16 (другие значения) |
| A + B | Удалить значение под курсором |
| L + ВЛЕВО/ВПРАВО | Предыдущая / следующая фраза |
| START | Воспроизведение фразы (зациклено) |

### Столбцы эффектов

Каждый слот FX состоит из двух частей: **типа** (3-буквенный код) и **значения** (2 шестнадцатеричных цифры). Используйте A+ВВЕРХ/ВНИЗ на типе для перебора доступных эффектов. Используйте A+ВВЕРХ/ВНИЗ на значении для его редактирования.

## 10. Экран INSTRUMENT (Инструмент)

Экран INSTRUMENT настраивает способ воспроизведения семпла. Переход: **R+ВПРАВО** из PHRASE, или **R+ВВЕРХ** из TABLE.

### Параметры

| Параметр | Диапазон | Описание |
|---|---|---|
| NAME | — | Название инструмента |
| SAMPLE | путь | Путь к WAV-файлу. Нажмите A для открытия файлового браузера. |
| ROOT | C-0 – B-9 | Высота звука семпла в оригинальной записи. Регулирует сдвиг высоты воспроизводимых нот. |
| DETUNE | 00–FF | Тонкая настройка. `80` = центр (без расстройки). Старший ниббл = целые полутоны (±8), младший = шаги 1/16 полутона. |
| VOL | 00–FF | Базовая громкость инструмента. `FF` = максимум. Перемножается с V-столбцом фразы и громкостями дорожки/мастера. |
| PAN | 00–FF | Стереопанорамирование. `00` = левый канал, `80` = центр, `FF` = правый канал. |
| START | 00–FF | Смещение начальной точки семпла (доля от полной длины). |
| END | 00–FF | Смещение конечной точки семпла. |
| LOOP | OFF / FWD / PNG | Режим петли: выкл, прямая петля, пинг-понг. |
| REV | OFF / ON | Обратное воспроизведение. |

### Цепочка громкости

Итоговая громкость каждой ноты:

```
VOL инструмента × Столбец V фразы × Громкость дорожки (Mixer) × Громкость мастера (Mixer)
```

### Навигация по инструментам

- **L + ВЛЕВО/ВПРАВО** — переключение между инструментами 00–FF
- **R+ВВЕРХ** из INSTRUMENT → экран MODULATION для данного инструмента
- **A** на поле SAMPLE → открывает файловый браузер

### Управление файловым браузером

| Ввод | Действие |
|---|---|
| ВВЕРХ/ВНИЗ | Перемещение по файлам/папкам |
| A | Выбрать файл (загрузить семпл) или войти в папку |
| B | Перейти на уровень выше |
| START | Предпрослушивание выделенного WAV-файла |

## 11. Экран TABLE (Таблица)

Таблица — это 16-строковый микросеквенсор, привязанный к инструменту. Когда нота звучит, таблица запускается параллельно с настраиваемой скоростью (скоростью тиков), применяя построчное транспонирование, громкость и эффекты поверх фразы.

Таблицы идеальны для: драм-роллов, глайдов нот, автоматических арпеджио, LFO-образной автоматизации.

Каждая строка:

```
     TRN  VOL  FX1      FX2      FX3
00   00   --   ---  00  ---  00  ---  00
01   03   --   ---  00  ---  00  ---  00
02   07   --   ---  00  ---  00  ---  00
...
```

| Столбец | Диапазон | Описание |
|---|---|---|
| TRN | 00–FF | Транспонирование в полутонах. `00` = без сдвига. Значения выше `7F` — отрицательные (например, `FC` = −4 полутона). |
| VOL | 00–FF / -- | Множитель громкости для строки. `--` = без изменений. |
| FX1–FX3 | как в фразе | Эффекты, применяемые на данном тике таблицы. |

### Скорость тиков (TIC)

В заголовке показано **TIC XX** — сколько тиков фразы проходит между строками таблицы. Используйте эффект **TIC** во фразе для изменения этого значения.

- По умолчанию: `0C` (12 тиков = одна строка таблицы за каждый шаг фразы)
- Меньше = таблица движется быстрее; больше = медленнее
- Специальные значения: `FC`=пинг-понг, `FE`=случайный порядок, `FF`=остановить

### Связь таблицы и инструмента

По умолчанию инструмент N использует таблицу N (например, инструмент `03` → таблица `03`). Переопределите для конкретной ноты с помощью эффекта **TBL** во фразе.

### Управление

| Ввод | Действие |
|---|---|
| Крестовина | Перемещение курсора |
| A + ВВЕРХ/ВНИЗ | Редактировать значение |
| A + B | Удалить значение |
| B + ВЛЕВО/ВПРАВО | Предыдущая / следующая таблица |
| L + B / копировать / вставить | Выделение, копирование, вставка (как в фразе) |
| START | Предпрослушивание инструмента с таблицей |

## 12. Экран GROOVE (Грув)

Экран GROOVE позволяет управлять длительностью каждого шага фразы по отдельности. Именно так создаётся **свинг**, **шаффл**, двойной темп, триоли и другие ритмические вариации.

Переход: **R+ВВЕРХ** из CHAIN или PHRASE (столбец 2).

Грув — это список до 16 значений тиков. Когда дорожка играет, она циклически проходит по грув-списку: первый шаг занимает столько тиков, сколько указано в строке 0 грува, второй — строка 1, и т. д. Когда список заканчивается, он возвращается к началу.

```
     TIC
00   0C    ← 12 тиков (стандарт)
01   --    ← пусто = конец списка, повтор с 00
```

### Пример свинга (грув 01)

```
     TIC
00   0E    ← 14 тиков (длинный)
01   0A    ← 10 тиков (короткий)
02   --    ← повтор
```

Это чередует длинные и короткие шаги — классический эффект свинга.

### Пример триолей

```
     TIC
00   08
01   08
02   08
03   --    ← повтор (3 × 8 = 24 тика за 3 шага, как 2 × 12)
```

### По умолчанию

Грув `00`, строка 0 = `0C` — все шаги ровно по 12 тиков (стандарт, без свинга).

### Назначение грувов

- По умолчанию каждая дорожка использует грув `00`.
- Используйте эффект **GRV XX** во фразе для переключения дорожки на грув `XX` начиная с этого шага.

### Управление

| Ввод | Действие |
|---|---|
| ВВЕРХ/ВНИЗ | Перемещение между строками |
| A + ВВЕРХ/ВНИЗ | Изменить значение тиков (+1/−1) |
| A + ВЛЕВО/ВПРАВО | Изменить значение тиков (+16/−16) |
| A + B | Очистить строку (установить --) |
| B + ВЛЕВО/ВПРАВО | Предыдущий / следующий грув |

## 13. Экран MODULATION (Модуляция)

Экран MODULATION (MODS) добавляет до **4 слотов модуляции** на инструмент. Каждый слот применяет автоматизированную огибающую или LFO к одному из параметров.

Переход: **R+ВВЕРХ** из INSTRUMENT (столбец 3).

### Типы модуляции

| Тип | Описание |
|---|---|
| `---` | Выкл (нет модуляции) |
| AHD | Одноразовая огибающая: Атака → Удержание → Спад |
| ADSR | Классическая огибающая: Атака → Спад → Сустейн → Отпускание |
| DRUM | Перкуссионная огибающая: резкий пик → тело → спад |
| LFO | Циклический осциллятор (треугольник, синус, пила, меандр, случайный, дрейф) |
| TRIG | AHD-огибающая, запускаемая другим инструментом/дорожкой |
| TRACKING | Маппинг высоты ноты или велосити на назначение (статическое значение) |

### Назначения модуляции

| Назначение | Воздействует на |
|---|---|
| VOLUME | Амплитуда |
| PAN | Стереопозиция |
| PITCH | Высота (в полутонах) |
| FINE | Высота (в центах, тонкое изменение) |
| CUTOFF | Частота среза фильтра |
| RES | Резонанс фильтра |
| SMPSTRT | Начальная точка семпла |
| MOD AMT | Глубина следующего слота модуляции |
| MOD RATE | Скорость следующего слота модуляции |
| MOD BOTH | Глубина и скорость следующего слота модуляции |

### Расположение

Экран показывает два слота бок о бок (MOD1+MOD2, затем MOD3+MOD4). Отображаемые параметры зависят от выбранного типа.

**Параметры AHD:**
- TYPE, DEST, AMT (глубина), ATK (тики атаки), HOLD (тики удержания), DEC (тики спада)

**Параметры ADSR:**
- TYPE, DEST, AMT, ATK, DEC, SUS (уровень сустейна), REL (тики отпускания)

**Параметры DRUM:**
- TYPE, DEST, AMT, PEAK (форма транзиента), BODY (тики удержания), DEC

**Параметры LFO:**
- TYPE, DEST, AMT, OSC (форма осциллятора), TRIG (режим триггера), FREQ (скорость)

**Режимы триггера LFO:**
- FREE — фаза никогда не сбрасывается (LFO работает непрерывно)
- RETRIG — фаза сбрасывается в 0 при каждой новой ноте
- HOLD — проигрывает один раз, удерживает последнее значение
- ONCE — проигрывает один цикл, возвращается в 0

**Формы LFO:** TRI, SIN, RAMP DN, RAMP UP, EXP DN, EXP UP, SQU DN, SQU UP, RANDOM, DRUNK
(Формы с суффиксом `_T` продвигаются по тикам, а не по шагам)

**Параметры TRACKING:**
- TYPE, DEST, SRC (NOTE или VELOCITY), LVAL (нижнее выходное значение), HVAL (верхнее выходное значение)

### Маршрутизация модуляция-в-модуляцию

Когда DEST установлен на **MOD AMT**, **MOD RATE** или **MOD BOTH**:
- Мод 1 модулирует Мод 2
- Мод 2 модулирует Мод 3
- Мод 3 модулирует Мод 4
- Мод 4 модулирует Мод 1 (круговой)

Пример: MOD1 (LFO) → MOD AMT → MOD2 (AHD): LFO ритмически раскачивает глубину огибающей.

### Управление

| Ввод | Действие |
|---|---|
| ВВЕРХ/ВНИЗ | Перемещение между параметрами |
| ВЛЕВО/ВПРАВО | Переключение между парными слотами (MOD1↔MOD2 или MOD3↔MOD4) |
| A + ВВЕРХ/ВНИЗ | Редактировать значение (малый шаг) |
| A + ВЛЕВО/ВПРАВО | Редактировать значение (большой шаг) |
| A + B | Сбросить на значение по умолчанию |
| B + ВЛЕВО/ВПРАВО | Предыдущий / следующий инструмент |

## 14. Экран MIXER (Микшер)

Экран MIXER показывает все 8 дорожек плюс мастер-канал с пиковыми измерителями уровня в реальном времени.

Переход: **R+ВНИЗ** из любого экрана 2-го ряда.

```
  T0   T1   T2   T3   T4   T5   T6   T7   MST
  ██   ██   ██   ██   --   --   --   --   ██
  ██   ██   ██   --                       ██
  ██   --   --
  80   80   80   80   80   80   80   80   80
```

Каждый столбец показывает пиковый измеритель (шкала dBFS) и текущее значение громкости (`00`–`FF`, где `80` = 0 дБ / единица).

### Шкала громкости

| Значение | Уровень |
|---|---|
| `00` | Тишина |
| `80` | Единица (0 дБ) |
| `FF` | Максимум (+6 дБ) |

### Зоны измерителя

| Цвет | Диапазон | Значение |
|---|---|---|
| Красный | ≥ 0 dBFS | Клиппинг |
| Жёлтый | от −6 до 0 dBFS | Горячий |
| Зелёный | ниже −6 dBFS | Норма |

### Управление

| Ввод | Действие |
|---|---|
| ВЛЕВО/ВПРАВО | Выбрать дорожку |
| A + ВВЕРХ/ВНИЗ | Увеличить / уменьшить громкость на 1 |
| A + ВЛЕВО/ВПРАВО | Увеличить / уменьшить громкость на 16 |

## 16. Справочник эффектов

Эффекты размещаются в столбцах **FX1**, **FX2** и **FX3** шага фразы, или в FX-столбцах строки таблицы. Каждый эффект имеет 3-буквенный код и 2-значное шестнадцатеричное значение.

Эффекты сохраняются до отмены (новой нотой на той же дорожке, новым эффектом в том же столбце или эффектом KILL), если не указано иное.

---

### ARP `XX` — Арпеджио

Быстрое чередование нескольких высот для имитации аккордов.

`XX` кодирует до двух интервальных смещений в полутонах:
- Старший ниббл (`X`) = первый интервал выше основного тона
- Младший ниббл (`Y`) = второй интервал выше основного тона

**Популярные значения:**

| Значение | Паттерн | Звучание |
|---|---|---|
| `37` | корень, +3, +7 | Минорный аккорд |
| `47` | корень, +4, +7 | Мажорный аккорд |
| `4B` | корень, +4, +11 | Мажорный септ |
| `3A` | корень, +3, +10 | Минорный септ |
| `CC` | корень, +12, +12 | Удвоение октавы |
| `00` | (отмена) | Остановить арпеджио |

Арпеджио сохраняется между шагами. Используйте `A00` для остановки. Настраивайте паттерн с помощью эффекта **ARC**.

---

### ARC `XX` — Конфиг арпеджио

Настраивает способ цикличности арпеджио.

- Старший ниббл (`X`) = режим: `0`=UP (вверх), `1`=DOWN (вниз), `2`=PINGPONG, `3`=RANDOM (случайный)
- Младший ниббл (`Y`) = скорость в тиках (`4` = по умолчанию)

Пример: `ARC 14` = режим DOWN, скорость 4 тика.

---

### CHA `XY` — Шанс (вероятность)

Ворота вероятности. При каждом воспроизведении шага генерирует случайное число.

- `X` (старший ниббл) = вероятность воспроизведения **левой части** этого FX-столбца (ноты + предшествующих FX). `0` = никогда, `F` = всегда.
- `Y` (младший ниббл) = вероятность воспроизведения **правой части** (последующих FX-столбцов).

Пример: `CHA F4` во FX2 = нота всегда звучит, FX3 срабатывает ~27% времени.

---

### DEL `XX` — Задержка

Задерживает шаг на `XX` тиков. Нота и все эффекты этой строки срабатывают на `XX` тиков позже обычного.

`DEL 00` = без задержки.

---

### GRV `XX` — Назначение грува

Переключает текущую дорожку на использование грува `XX` начиная с этого шага.

Смотрите [Экран GROOVE](#12-экран-groove-грув) для редактирования грувов.

---

### HOP `XY` — Прыжок

Перемещает воспроизводящую головку в другую позицию.

- Во **фразе**: переходит к шагу `Y` фразы, ограничено `X` раз. После `X` прыжков — сквозное воспроизведение.
- В **таблице**: переходит к строке `Y` таблицы.

`HOP 00` в конце секции фразы = зациклить эту секцию бесконечно.

---

### KIL `00` — Стоп

Немедленно останавливает семпл на этой дорожке. Отменяет все сохраняющиеся эффекты (ARP, REP, эффекты высоты).

---

### OFF `XX` — Смещение

Переносит начальную точку воспроизведения семпла к смещению `XX` (доля от полной длины). Удобно для обращения к разным частям длинного семпла.

---

### PSL `XX` — Питч-слайд (Портаменто)

Скользит высотой от предыдущей ноты к текущей за `XX` тиков.

`PSL 00` = мгновенная смена высоты (без слайда).

---

### PBN `XX` — Питч-бенд

Непрерывный сдвиг высоты. Сохраняется до отмены.

- `00`–`7F` = бенд ВВЕРХ (выше значение = быстрее)
- `80`–`FF` = бенд ВНИЗ (значения выше `80` = быстрее вниз)
- `PBN 00` = отмена

---

### PVB `XY` — Вибрато

Колеблет высоту вверх и вниз.

- `X` = скорость (`0`–`F`, выше = быстрее, диапазон ~2–9,5 Гц)
- `Y` = глубина (`0`–`F`, выше = шире, до ~1,9 полутона)

Сохраняется до `PVB 00` или новой ноты или KILL.

---

### PVX `XY` — Экстремальное вибрато

То же, что PVB, но в 4× глубже и в 2× быстрее. Для драматичного вобблинга и LFO-эффектов высоты.

---

### REP `XY` — Повтор (Ретриггер)

Повторный триггер ноты внутри шага.

- **Y = 0 (режим простого ретриггера):** Ретриггер каждые `X` тиков. `R30` = каждые 3 тика.
- **Y ≠ 0 (режим ступенчатой громкости):** Ретриггер каждые `Y` тиков с изменением громкости при каждом ударе.
  - `X` = 0: без изменения громкости
  - `X` = 1–7: уменьшать громкость при каждом ретриггере (1=слабо, 7=агрессивно)
  - `X` = 8–F: увеличивать громкость (эффект нарастания)

Сохраняется между шагами. Отмена: новая нота, новый FX в том же столбце или KILL.

---

### RND `XY` — Рандомизация

Рандомизирует **предыдущее активное FX-значение** на этой дорожке.

- `X` = диапазон рандомизации вниз (`0`=нет, `F`=максимум)
- `Y` = диапазон рандомизации вверх

---

### RNL `XY` — Рандомизация влево

Рандомизирует FX-значение **в столбце непосредственно слева**.

- `RNL` во FX1 = рандомизирует высоту ноты и инструмент
- `RNL` во FX2 = рандомизирует значение FX1
- `RNL` во FX3 = рандомизирует значение FX2

`X`/`Y` — те же диапазоны, что и у RND.

---

### TBL `XX` — Назначение таблицы

Переопределяет стандартную таблицу инструмента, используя таблицу `XX` для данной ноты.

---

### THO `XX` — Прыжок в таблице

Перемещает воспроизводящую головку таблицы к строке `0X`. Работает как HOP, но для строк таблицы.

`THO 00` в конце секции таблицы = зациклить эту секцию.

---

### TIC `XX` — Скорость тиков

Устанавливает скорость тиков таблицы (сколько тиков фразы на одно продвижение строки таблицы).

- `TIC 0C` = 12 тиков на строку (одна строка на шаг фразы — по умолчанию)
- `TIC 06` = 6 тиков на строку (таблица движется вдвое быстрее)
- `TIC FC` = пинг-понг (таблица двигается туда-обратно)
- `TIC FE` = случайная строка при каждом тике
- `TIC FF` = остановить таблицу

---

### VOL `XX` — Автоматизация громкости

Перезаписывает громкость шага значением `XX` в момент срабатывания команды. Можно использовать в строках таблицы для анимации громкости во времени.

---

## 17. Справочник модуляции

Смотрите [Экран MODULATION](#13-экран-modulation-модуляция) для редактирования слотов модуляции.

### Огибающая AHD

Одноразовая огибающая, запускаемая при каждой ноте.

```
     ┌──────┐
     │      │
  ATK│ HOLD │DEC
     │      └────
─────┘            ─ (тишина)
```

- **ATK** (`00`–`FF`) — время атаки в тиках (0 = мгновенно)
- **HOLD** (`00`–`FF`) — время удержания пика
- **DEC** (`00`–`FF`) — время спада до тишины

### Огибающая ADSR

Классическая синтезаторная огибающая. Отпускание запускается при окончании ноты (KILL или перехвате голоса).

- **ATK** — время атаки
- **DEC** — спад до уровня сустейна
- **SUS** (`00`–`FF`) — уровень сустейна (00 = тишина, FF = полный)
- **REL** — время отпускания после окончания ноты

### Огибающая DRUM

Для перкуссионных звуков. Резкий пик, короткое тело, долгий спад.

- **PEAK** — форма начального транзиента
- **BODY** — время удержания после пика
- **DEC** — время спада хвоста

### LFO

Циклическая модуляция. Скорость задаётся параметром **FREQ**.

- **OSC** — форма (TRI, SIN, RAMP DN, RAMP UP, EXP DN, EXP UP, SQU DN, SQU UP, RANDOM, DRUNK)
- **TRIG** — FREE / RETRIG / HOLD / ONCE
- **FREQ** (`00`–`FF`) — скорость (выше = быстрее)

Формы с суффиксом `_T` используют тики как базу времени (ощущение синхронизации с темпом); прочие — музыкальные шаги.

### Огибающая TRIG

Как AHD, но запускается другим инструментом или дорожкой, а не собственной нотой.

- **SRC** — источник: инструмент `00`–`7F` или дорожка `80`–`87`
- ATK, HOLD, DEC — как в AHD

### TRACKING

Считывает высоту ноты или велосити и маппирует их на назначение.

- **SRC** — NOTE (высота), VELOCITY (велосити), или VEL_TAKEOVER
- **LVAL** — выходное значение при минимальном источнике
- **HVAL** — выходное значение при максимальном источнике

Пример: TRACKING NOTE → CUTOFF, LVAL `20`, HVAL `E0` = низкие ноты — тёмный фильтр, высокие — яркий.

- Формат: `.ptp` (внутри JSON)
- Путь по умолчанию: `/Documents/PocketTracker/Projects/`
- Сохранение: экран PROJECT → SAVE
- Загрузка: экран PROJECT → LOAD

### Файлы семплов

- Формат: `.wav` (моно или стерео; стерео автоматически конвертируется в моно при загрузке)
- Частота дискретизации: любая — PocketTracker автоматически компенсирует высоту для семплов не 44100 Гц
- Загрузка: экран INSTRUMENT → поле SAMPLE → кнопка A → файловый браузер

### WAV-экспорт

- Формат: 16-бит стерео WAV, 44100 Гц
- Путь: `/Documents/PocketTracker/Renders/`
- Имя файла: `ProjectName_0001.wav`, `_0002.wav`, … (автоинкремент)
- Запуск: экран PROJECT → WAV MIX

### Ресемплированные инструменты

Создаются через **ресемплинг выделения** (см. ниже):
- Формат: 16-бит стерео WAV
- Путь: `/Documents/PocketTracker/Samples/Resampled/`
- Имя файла: `Resample_0001.wav`, … (автоинкремент)

### Ресемплинг выделения

Вы можете отрендерить выделение дорожек из экрана SONG прямо в новый слот инструмента:

1. На экране SONG войдите в режим выделения (**L + B**).
2. Выделите строки и дорожки, которые хотите отрендерить.
3. **Двойное нажатие A** — появляется диалог подтверждения.
4. Выберите **YES** — выделенные дорожки рендерятся офлайн в WAV-файл.
5. Новый инструмент создаётся в первом свободном слоте, загруженный отрендеренным аудио.
6. Строка статуса показывает номер нового инструмента (например, `RESAMPLED TO INST 0C`).

## 19. Советы по работе

### Создание первой фразы

1. Откройте экран **PHRASE** (R+ВПРАВО из SONG).
2. Курсор на строку 00, столбец N (Нота).
3. Нажмите **A** — вставляется нота по умолчанию (C-4, инструмент 00).
4. Используйте **A+ВВЕРХ/ВНИЗ** для изменения высоты.
5. Используйте **A+ВПРАВО/ВЛЕВО** для изменения октавы.
6. Переместите курсор вправо к столбцу **I**, используйте **A+ВВЕРХ/ВНИЗ** для выбора инструмента.
7. Нажмите **START** для прослушивания фразы в цикле.

### Создание базового бита

1. Загрузите бочку на инструмент `00` (по умолчанию или свой WAV).
2. Загрузите малый барабан на `01`, хэт на `02`.
3. В фразе `00`: разместите бочку на шагах 00, 04, 08, 0C (каждые 4 шага).
4. Разместите малый барабан на шагах 04, 0C.
5. Разместите хэт на шагах 00, 02, 04, 06, 08, 0A, 0C, 0E (каждые 2 шага).
6. Создайте цепочку (`00`), указывающую на фразу `00`.
7. В SONG поставьте цепочку `00` на дорожку 0, строку 00.
8. Нажмите START на экране SONG.

### Использование транспонирования в цепочках

Вместо дублирования фразы на другой высоте, добавьте значение транспонирования в цепочке:
- В CHAIN установите столбец **TRN** в `07` для +7 полутонов (квинта выше).
- Та же фраза звучит выше без какого-либо копирования.

### Свинг через грув

1. Перейдите на экран GROOVE (R+ВВЕРХ из PHRASE).
2. Создайте грув `01` со строками: `0E`, `0A`, `--`.
3. В фразе добавьте `GRV 01` в столбец FX шага 00.
4. Дорожка теперь играет со свинговым ритмом.

### Драм-ролл через таблицу

1. В таблице `03` (используется инструментом 03 — малый барабан), установите строки 00–07: TRN `00`, FX1 `REP 01`.
2. Это заставит малый барабан ретриггериться на каждом тике таблицы.
3. Установите низкую скорость тиков (`TIC 03` во фразе) для быстрого ролла.

### Модуляция: Нарастание громкости пэда

1. Откройте INSTRUMENT для вашего пэда, затем перейдите на MODS (R+ВВЕРХ).
2. Установите MOD1 TYPE = `AHD`.
3. Установите MOD1 DEST = `VOLUME`.
4. Установите ATK = `40`, HOLD = `20`, DEC = `60`.
5. Каждый раз, когда инструмент срабатывает, огибающая формирует амплитуду.

### WAV-экспорт

1. Создайте полную песню.
2. Перейдите на экран PROJECT.
3. Переместите курсор на `WAV MIX` и нажмите **A**.
4. Песня рендерится офлайн. Строка статуса покажет имя выходного файла после завершения.

## Приложение А: Краткий справочник шестнадцатеричных чисел

PocketTracker использует шестнадцатеричную (base-16) систему для большинства значений:

| Десятичное | Шест-е | Десятичное | Шест-е |
|---|---|---|---|
| 0 | `00` | 128 | `80` |
| 16 | `10` | 160 | `A0` |
| 32 | `20` | 192 | `C0` |
| 64 | `40` | 224 | `E0` |
| 96 | `60` | 255 | `FF` |

Половина от `FF` (максимум) = `80` (центр/единица). Это значение по умолчанию для VOL и PAN.

## Приложение Б: Названия нот

```
C  C# D  D# E  F  F# G  G# A  A# B
До Ре* Ре Ми* Ми Фа Соль* Соль Ля* Ля Си* Си
00 01 02 03 04 05 06 07 08 09 0A 0B  (смещение в полутонах внутри октавы)
```

Средняя C = `C-4` = MIDI-нота 60.

Полный диапазон: `C-0` (самая низкая) до `B-9` (самая высокая).

## Приложение В: Инструменты по умолчанию (00–0B)

Первые 12 слотов инструментов (00–0B) загружаются из встроенных семплов:

| Слот | Семпл |
|---|---|
| 00 | Бочка (Kick) |
| 01 | Малый барабан (Snare) |
| 02 | Закрытый хэт (Hihat closed) |
| 03 | Открытый хэт (Hihat open) |
| 04 | Бас (Bass) |
| 05–0B | Дополнительные стандартные семплы |

Слоты `0C`–`FF` — пользовательские.
